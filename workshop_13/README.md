# **ПРАКТИЧНА РОБОТА №13**

## **Тема: Дослідження кольорових гармоній та інструментів аналізу кольору в Adobe Color**
## **Мета:**
1. Ознайомити студентів із поняттям кольорової гармонії та основними типами
гармонії кольорів.
2. Навчити студентів використовувати колірне колесо для побудови палітр за різними
принципами гармонії.
3. Ознайомити із можливостями автоматичного виділення кольорових палітр із
зображень та створення градієнтів.
4. Навчити перевіряти контрастність кольорових пар відповідно до стандартів
доступності (WCAG).
5. Закріпити навички документування та аналізу роботи з кольором у
Markdown-форматі у GitHub-репозиторії.

## **ВИСНОВОК:**
За результатами дослідження кольорових гармоній та практичної роботи з інструментами Adobe Color можна зробити такі ключові висновки щодо ефективного використання кольору в дизайні інтерфейсів (UI/UX):

1. Вибір Гармонії для ІнтерфейсуДля
   
- Основи та Читабельності: Найкраще підходять Монохроматична та Аналогічна гармонії. Вони створюють візуальну єдність, мінімальний когнітивний дисонанс і забезпечують спокійний фон для контенту, що є критичним для довготривалого використання додатків.
- Для Акцентів: Комплементарна (особливо Розділена Комплементарна) та Тріада гармонії ідеально підходять для кнопок Call to Action (CTA), сповіщень та елементів, які вимагають максимальної уваги. Однак їх слід використовувати економно, щоб не перевантажувати користувача.

2. Оптимальний Підхід до Палітри (Extract Theme)
   
- Підхід Extract Theme довів, що для створення основи професійного інтерфейсу перевагу слід надавати режиму Muted (Приглушений). Приглушені кольори забезпечують нейтральність та кращу читабельність, тоді як насичені кольори (режим Colorful) повинні бути зарезервовані виключно для акцентів, щоб не відволікати від функціоналу.

3. Критична Важливість Доступності (WCAG)
   
- Контраст є пріоритетом: Практичний аналіз контрасту показав, що естетично приємні пари кольорів можуть бути недоступними. Наш приклад (#FFC745 на #007A78) з коефіцієнтом $3.33:1$ не пройшов вимоги WCAG AA для звичайного тексту.Необхідність Корекції: Використання інструментів доступності (Accessibility Tools) для корекції є обов'язковим. Зміна фону дозволила досягти коефіцієнта $13.51:1$, що значно перевищує вимоги WCAG AAA ($7:1$), гарантуючи максимальну інклюзивність та зручність читання для всіх користувачів.
